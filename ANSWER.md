**4. Скорость и параметры модели**

По оптимизации скорости — проверил детально, вот что происходит сейчас и что можно сделать:

Сейчас в RAM одновременно загружены три модели:
- `qwen3.6-35b-a3b` (20 ГБ) — основная для проверки отчётов
- `qwen3-coder-30b-a3b` (18 ГБ) — для работы с кодом
- `qwen3-4b` (2.4 ГБ) — лёгкая модель

Итого они занимают ~156 ГБ из 503 ГБ RAM, остаётся свободно ~311 ГБ.

Ключевая проблема: LM Studio по умолчанию обрабатывает запросы **строго последовательно** (один за одним). Сейчас проверка отчёта — это 12 разделов, каждый ждёт, пока завершится предыдущий. Именно поэтому два параллельных отчёта у тебя фактически проверялись один за другим.

На GPU всё работало бы иначе — там запросы можно батчить и выполнять одновременно. На CPU каждый дополнительный параллельный слот просто делит процессорное время, суммарная скорость почти не растёт. Поэтому правильный путь — запустить **несколько независимых экземпляров** модели на разных портах. RAM позволяет держать одновременно до ~12 копий qwen3.6-35b-a3b (по 20 ГБ каждая = 240 ГБ, влезает). Тогда 12 разделов одного отчёта уйдут параллельно — каждый в свой экземпляр — и время проверки сократится примерно в 10–12 раз.

Буду постепенно наращивать этот параллелизм, начну с 2–3 экземпляров чтобы не перегружать серверные CPU. Видеокарта RTX с 24+ ГБ VRAM дополнительно ускорила бы каждый отдельный запрос в 10–20 раз.
