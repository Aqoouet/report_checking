# Report Checker

Сервис автоматической проверки технических отчётов (.docx и .pdf) с помощью локальной нейросети через LM Studio. Пользователь вводит путь к файлу на сервере — система прогоняет документ через набор проверок и формирует текстовый отчёт об ошибках.

## Проверки

| Критерий | .docx | .pdf |
|---|:---:|:---:|
| Ясность изложения и научно-технический стиль | ✓ | ✓ |
| Представление физических величин и единиц измерения | ✓ | ✓ |
| Перекрёстные ссылки на таблицы и рисунки | ✓ | — |

Каждый критерий — отдельный «чекпоинт», результаты которого отображаются в реальном времени. Проверку можно остановить досрочно — частичный отчёт будет доступен для скачивания.

## Быстрый старт (Docker — рекомендуется)

Предполагается, что LM Studio уже запущен на хосте на порту `1234`.

```bash
cd report_checking

# 1. Скопируйте конфиг и заполните имена моделей
cp .env.example .env
# Узнать имена загруженных моделей:
#   curl http://localhost:1234/v1/models
# Заполните OPENAI_MODEL и OPENAI_VALIDATE_MODEL

# 2. Соберите и запустите
docker compose up --build -d
```

Откройте http://localhost:5173

## Разработка без Docker

### Бэкенд

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
# Установите OPENAI_BASE_URL=http://localhost:1234/v1

uvicorn main:app --reload --port 8000
```

### Фронтенд

```bash
cd frontend
echo "VITE_API_URL=http://localhost:8000" > .env
npm install
npm run dev
```

Откройте http://localhost:5173

## Переменные окружения (`.env` / `backend/.env`)

| Переменная | Описание | Значение по умолчанию |
|---|---|---|
| `OPENAI_API_KEY` | Любая строка (LM Studio не проверяет) | `lm-studio` |
| `OPENAI_BASE_URL` | Адрес сервера LM Studio | `http://localhost:1234/v1` |
| `OPENAI_MODEL` | Имя модели для проверок | `qwen3-coder-30b-a3b-instruct` |
| `OPENAI_VALIDATE_MODEL` | Модель для валидации диапазона (пусто = OPENAI_MODEL) | `qwen/qwen3-4b-2507` |
| `AI_TIMEOUT` | Таймаут ответа модели (секунды; 0 = без ограничений) | `0` |
| `AI_CONNECT_TIMEOUT` | Таймаут подключения к серверу (секунды) | `15` |
| `AI_CHUNK_MAX_TOKENS` | Лимит токенов для ответа на один раздел/страницу (0 = по умолчанию сервера) | `0` |
| `APP_PORT` | Порт веб-интерфейса | `5173` |
| `LO_FIELD_UPDATE` | Использовать LibreOffice для обновления полей .docx (0 = отключить) | `1` |

## Как работает

1. Пользователь вводит путь к файлу (.docx или .pdf) и, опционально, диапазон разделов или страниц.
2. Путь маппируется из Windows-формата в Linux (настраивается в `backend/path_mapping.json`).
3. Для .docx: LibreOffice обновляет поля (STYLEREF/SEQ/REF), затем `python-docx` извлекает листовые разделы с текстом.
4. Для .pdf: PyMuPDF извлекает текст постранично.
5. Каждый включённый чекпоинт последовательно обрабатывает разделы/страницы, отправляя их нейросети.
6. Пользователь скачивает итоговый текстовый отчёт (`report_errors.txt`).
